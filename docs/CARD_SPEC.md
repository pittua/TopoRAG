# 知識カード 仕様（CARD_SPEC）

LLM 合成層が「近縁回路を弁別」するための知識を載せる **知識カード** の契約。
実体は `knowledge_cards.yaml`（回路 id をキーとする）。

方針再定義（2026-06-11）に基づく設計思想:

> RAG コーパスは役割説明ではなく、**識別の決め手・紛らわしい近縁との差分**を載せる。
> 取り違えログが知識の種である。

つまりカードは「この回路は何か」を長々と説明するものではない（教科書回路の役割は
LLM が事前学習済み）。**「IR のどこを見ればこの回路だと確定できるか」**と
**「よく似た別回路とどこが違うか」**だけを、知覚層が出す IR フィールドに紐づけて書く。

## 1 枚のスキーマ

`knowledge_cards.yaml` の `cards` 直下に、回路 id をキーとして並べる。

```yaml
cards:
  <circuit_id>:               # 検索ヒット／DB エントリの id と一致（必須）
    name: <回路名>             # 必須
    role: <一文の役割>          # 必須。事前学習済み知識の呼び水。冗長な説明は書かない
    decisive:                  # 必須。識別の決め手（list[str]）
      - <IR フィールドに紐づく観察可能な手がかり>
    confused_with:             # 任意。紛らわしい近縁との差分（list）
      - with: <別の circuit_id> # DB に実在する id（テストで解決を検査）
        difference: <IR 上で何がどう違うか＝弁別の決め手>
    pitfalls: <任意。取り違えやすい点・限界（取り違えログ由来）>
```

### フィールド規約

| フィールド | 必須 | 規約 |
|---|---|---|
| `name` | ○ | 回路名（DB の name と一致させる） |
| `role` | ○ | **一文**。役割の長文説明は禁止（DB の description が担う） |
| `decisive` | ○ | 各項は「IR の `diode.series=true` のような観察可能な根拠」を含める。LLM が IR と照合できる粒度で書く |
| `confused_with` | × | `with` は **実在の circuit_id**。`difference` は両者の IR 差分を 1〜2 文で |
| `pitfalls` | × | トポロジーのみでの限界・既知の誤マッチなど |

## 執筆原則

1. **決め手は IR フィールド名で裏づける。** 「半波整流はダイオードが直列
   （`diode.series=true`, `diode.anode_at_input=true`）」のように、知覚層が
   実際に出す事実に紐づける。机上の一般論で書かない（IR で照合できなくなる）。
2. **差分は取り違えログから書く。** `confused_with` は実評価で衝突・誤マッチした
   ペアを優先する（`docs/`・メモ `toporag-eval-findings` の取り違え）。自明に
   分離する遠縁同士は書かない。
3. **役割説明を増やさない。** カードの価値は弁別であって解説ではない。

## 後続との関係

- **③プロンプト再設計**: system 文「カードと IR を照合して裁定せよ」で、検索 top-3 の
  カードを根拠として提示する。
- **④3アーム評価**: アームC(+IR+RAG) がこのカード top-3 を注入する。アームB(+IR) との
  差が「弁別知識の寄与」を測る。
