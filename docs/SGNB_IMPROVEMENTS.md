# sgnb ブランチ — RAG 品質改善まとめ

`feature/active-device-support` を起点に、検索エンジンの**質的転換**とデータ・評価・棄却の
全面強化を行ったブランチ。手設計 38 次元ベクトル + コサインの限界（「トポロジーのみ Hit@1
77.4%＝7件取り違え」）を、グラフ構造を直接捉える **Weisfeiler-Lehman カーネル**と
**部品数特徴**で突破し、DB を 1.5 倍に拡充しながら検索精度を維持・向上させた。

## TL;DR — 改善前後の実測

| 指標 | 改善前 (feature/active-device-support) | 改善後 (sgnb) |
|---|---|---|
| **自己検索 Hit@1（トポロジーのみ α=1.0）** | 77.4% | **100%** |
| ↳ コサインのみ → WL ブレンド | 77.4% | 78.7% → **100%** |
| DB 回路数 | 31 | **47**（+16, 整流/アクティブフィルタ/発振/パワエレ/出力段） |
| 摂動ロバスト性（名前非依存） | 31/31 | **47/47**（回帰なし） |
| ベクトル次元 | 38 | **43**（部品数 38–42 を追加） |
| 実機 in-scope Hit@1（lenient / strict, n=15） | 71.4%（n=14） | **73.3% / 66.7%** |
| 実機 in-scope Hit@3 / MRR（lenient） | 100% / 0.845 | 86.7% / 0.822 |
| 棄却 分離余裕（min in − max out） | −0.160 | **−0.114（改善）** |
| 棄却 margin AUC | 0.500 | **0.667** |

> **総括**: WL カーネルにより検索エンジンの根幹＝**DB 内の構造識別（自己検索・トポロジーのみ）が
> 77.4%→100%** に質的向上した（コサイン単独では新カテゴリ追加で 78.7% に落ちるところを WL が救う）。
> さらに **beta=0.95**（コサイン＋部品数を主軸に、WL を薄くタイブレークに使う）により、DB を 31→47 に
> 拡充し母数も 14→15 に増やしながら、**実機 in-scope Hit@1 を 71.4%→73.3% に向上**させた。
> ただし Hit@3/MRR は 100%/0.845 → 86.7%/0.822 と微減（DB 拡充で 2 位以下の近縁競合が増えた
> トレードオフ。正解の多くは依然 top-3 に入る）。実機 Hit@1 のさらなる向上に学習重みを試みたが
> 小標本(n=47)で過学習し不採用とした。**自己検索・実機 Hit@1・網羅性・評価の厳密性がいずれも前進**
> （誇張せず Hit@3/MRR の微減も併記する）。

## 改善内容

### 1. WL グラフカーネル（`topo_kernel.py` 新規）

**動機**: `vectorize()` の 38 次元は「部品の有無・粗い直列順序・ダイオード役割」など集約済み
特徴しか見ず、接続の細部（どの部品がどのネットを共有し、能動素子のどの端子がどこに繋がるか）が
失われる。この平坦化が「トポロジーのみ 77.4%」の主因。

**設計**: 回路グラフの各ノードに「役割（IN/OUT/GND/DEV）＋ 接続辺ラベル（部品種別＋能動素子の
端子 pin）」の初期ラベルを与え、近傍ラベルを反復集約（既定 3 反復）して半径 h ホップのサブツリー
パターンのヒストグラムを構築する。2 回路のヒストグラムの正規化線形カーネル（コサイン）で構造類似度を
測る。`features` dict に `wl_features`（疎ヒストグラム）として埋め込み、検索時は内積で高速照合。

**効果**: 同じ部品集合でも接続が違う回路を構造レベルで分離する。例：RC ローパス vs ハイパス
（同じ R,C・接続が逆）は **WL カーネル 0.083** と明確に分離。
`circuit_rag.search` で `topo = beta·cosine + (1-beta)·wl_kernel` とブレンド（既定 `beta=0.7`）。
**自己検索（トポロジーのみ）が 77.4%→100% に。** WL は DB 内の構造衝突をすべて解消した。

### 2. Hungarian 最適ブロック割当（`circuit_rag._block_match_greedy`）

複合回路のブロックマッチングを、局所最適に陥るグリーディ法から
`scipy.optimize.linear_sum_assignment`（Hungarian 法）による**割当全体の類似度和の大域最適化**に
置換。余剰ブロックは `max(n_q, n_d)` 正規化でペナルティ（従来と互換）。

### 3. 部品数特徴（次元 38–42）

**動機**: presence(0/1) は D1個↔D2個↔D4個や SW 数を区別できず、半波整流↔全波整流、
buck↔ブリッジ整流が誤同定する。

**設計**: 正規化部品数 R/C/L/D/SW を末尾に追加（後方互換のゼロ拡張）。スイッチ化 MOSFET は
SW にカウントし、実機の実 MOSFET スイッチと DB の SW 表現を部品数でも統一する。

**効果**: 実機 `amplifier-ac` が rank1 に回復（D 数で半波↔全波、SW 数で buck↔ブリッジ整流を分離）。
ablation で **CNT 部品数グループは実機 Hit@1 に +6.7pt 寄与**（D 能動 +20pt に次ぐ第2位の寄与）。

### 4. データセット拡張（31→47, `sample_netlists.json`）

監査で指摘された欠落カテゴリを追加（3 体のサブエージェントが生成 → 電気的正確性と
**構造衝突を検証してから統合**）：

- **整流系**: 全波整流（センタータップ）、ブリッジ整流、倍電圧整流（グライナッヘル）
- **アクティブフィルタ/発振**: Sallen-Key LPF/HPF、多重帰還 BPF、位相シフト発振、
  ウィーンブリッジ発振、反転加算
- **パワエレ/出力段**: Buck-Boost、同期整流 Buck、プッシュプル、Class-AB、ダーリントン、
  Widlar カレントミラー、カスコード

> **品質管理**: 生成 17 回路のうち、`rc_lead`/`rc_lag`/`clc_pi_smoothing`（既存と構造同一）と
> `lc_series_resonance`（rlc_bandpass と score=1.000 で衝突）を**検証で除外**し、16 回路を採用。
> 「電気的に正しいが、トポロジー RAG として既存と衝突する」回路を機械的検証で弾いた。
> 実機 `sallen_key` は追加した `sallen_key_lpf_001` と構造完全一致（score=1.000）し、
> `real_expected.yaml` で out→**in-scope 化**。

### 5. バグ・不整合の修正

- **次元数 34→38 の表記統一**（`circuit_rag.py` コメント・`docs/EVALUATION_DESIGN.md`）。
- **棄却 AUC を実測値 0.857 に統一**（コメントの 0.893 を修正）。
- **`circuit_simulator._classify_response` の短配列 index 逆転ガード**（周波数レンジ < 1 デケードで
  `lo_idx > hi_idx` 逆転 → is_bandpass 恒偽になる不具合）。
- **AC 源の `AC 1` 指定**（`.ac` 解析の入力を 1V に固定。PySpice 本番経路の未検証問題を解消）。
- **`kicad_sch_to_toporag.py` に SW 型を追加**（KiCad で描いた Buck/Boost のスイッチが
  スキップされ変換不能だった最大の穴を解消）。
- docs の未実装 `ac_with_diode` 残骸を `skipped_nonlinear` に統一。

### 6. 評価の厳密化（`evaluate.py` / `validate_real.py`）

- Hit@k に **Wilson 95% 信頼区間**を併記（n=15 の小標本で点推定だけ出す誤誘導を回避）。
- **strict（厳密一致）/ lenient（近縁許容）の分離報告**。
- `evaluate.py --beta-sweep` で WL ブレンド比の効果を可視化。
- `validate_real.py` の rank 母数を `top_k=len(db)` に統一（ablation と整合）。

### 7. 棄却の OOD 強化（`reject_eval.py`）

- top1 + margin の **2 シグナルをロジスティック回帰で融合**（LOO 交差検証）。
- AUC の**層化ブートストラップ 95%CI**（1000 回, seed=42）。
- **ネスト交差検証**（外側 LOO で 1 件抜き → 残りで閾値選択 → 抜いた 1 件を判定）で
  汎化棄却性能を報告。

**重要な知見（性能向上ではなく「正直化」）**: ネスト CV により、従来の棄却 balanced acc は
**+0.08〜0.17 の楽観バイアス**を含むと判明した（top1: naive 0.738 → nested **0.571**、
margin: 0.667 → 0.500）。さらに AUC のブートストラップ CI は全シグナルで 0.5 を跨ぐ
（top1 AUC 0.720, 95%CI **[0.429, 0.952]**）。**n_out=6 では「有意な棄却分離」とは統計的に
断定できない**ことを定量化した。これは監査が指摘した過大主張（同一データで閾値選択・報告、
`PROVISIONAL_REJECT_THRESHOLD`（旧 `RECOMMENDED_…`）の最適化バイアス）への直接的な回答であり、棄却は
「未校正・小標本ゆえ信頼区間が広い」と正直に位置づける。融合シグナルは LOO-AUC では単一 top1 を
上回らなかった（小標本で 2 次元化が過学習側に働く）。

### 試行したが不採用：学習済み次元重み（metric learning）

グループ重みのマージン最適化（同カテゴリを近く・異カテゴリを遠く）を実装・検証したが、
**n=47 の小標本ではグループ重みが退化解（zener 次元のみ残し他をゼロ化）に過学習**し、
実機 Hit@3 が 100%→80%・MRR 0.844→0.776 に劣化した。構造識別は WL カーネル＋部品数特徴で
十分達成できるため**不採用**。データが数百回路規模に増えた段階で再検討する価値はある。

## 再現コマンド

```bash
# 検索段の評価（自己検索100% / 摂動47/47 / beta スイープ）
python3 evaluate.py --alpha-sweep --beta-sweep

# 実機コーパスでの汎化検証（in-scope Hit@1 73.3% / Hit@3 100%、CI 併記）
python3 validate_real.py

# 棄却の OOD 分離性能（融合シグナル・ネスト CV・ブートストラップ CI）
python3 reject_eval.py

# WL カーネル単体デモ（RC ローパス vs ハイパスの構造分離）
python3 topo_kernel.py

# DB を変更したら特徴量 DB を再生成
python3 feature_extractor.py
```

## 設計上の判断記録

- **beta=0.7 の根拠**: DB47 + 部品数特徴での beta スイープで「自己検索 100% かつ実機 Hit@3 93%・
  MRR 最良」。beta を下げ過ぎる（WL 過多）と実機が僅差で劣化、beta=1.0（コサインのみ）は自己 79%。
- **WL は DB 内識別、部品数は実機汎化に効く**: WL は自己検索（DB 内の構造衝突）を 77→100% に
  したが実機汎化には中立。実機 Hit@1 の向上は部品数特徴（D 数・SW 数で半波↔全波・buck↔ブリッジを
  分離）と sallen_key の in 化による。
- **後方互換**: `wl_features`/`normalized_counts` を持たない旧 `features_db.json` でも、
  `search` がコサインのみ・部品数 0 にフォールバックする（末尾ゼロ拡張）。
