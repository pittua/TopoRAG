## 0. 一行サマリ（強さを明示）

- 「**GraphRAG × 回路ネットリスト**」の看板研究は確かに存在する（AMSnet-KG, `[要旨確認]`）。読んだ範囲では**設計＝生成側が中心**。
- **認識（understanding）側**でも「階層グラフ＋LLM 推論」が最近の主流（TopoSizing `[要旨確認]`, HeaRT `[要旨確認]`）。
- TopoRAG の **(a) 認識特化**、**(b) OOD 棄却**、**(c) 決定的 IR 重視** は、**私が読んだ数本の中では**空きに見える。ただし後述の通り **これは「空きの証明」ではない**（§3.2, §5）。
- TopoRAG の **フラット最近傍検索**は、読んだ主流（サブ回路単位の構成的認識）より素朴。ただし「構成的にすれば解決」とも言い切れない（HeaRT は難回路で苦戦、§3.3）。
- **【重要・2026-07-08 追記】「構造を特徴量として回路を分類する」という枠組みそれ自体は新規性がない**。GCN/GNN による回路分類・サブ回路認識は成熟領域で、貴手法の検索段（43 次元手設計特徴＋最近傍）と直接競合する（§2.7）。したがって**発表で「分類」を新規性の核に据えてはならない**。さらに実験A（`RESULTS_experiment_A_2026-07-05.md`）では直読み LLM の out 棄却率が提案手法を上回った（95.0% > 90.0%）ため、**「OOD 棄却できること」も差別化にならない**。生き残る主張は「監査可能・再現可能・根拠（IR）付きの認識」という運用面の非対称のみ（§3.2）。

## 2. 各研究の詳細

各項末尾に「TopoRAG への含意」を**肯定面と否定面の両方**で記す（確証バイアス対策）。

### 2.1 AMSnet-KG（arXiv:2411.13560, 提出 2024-11-07）`[要旨確認]` — 「KG-RAG×回路」の看板
- **問題**: 公開 AMS データ希少で LLM は正確な netlist を直接生成できない。要旨は「仕様→netlist の自動設計フロー」を明言 `[要旨確認]`。
- **方法**`[本文のみ]`: 回路オブジェクト＋文字列エンティティを **entity-relation-entity** で結ぶ知識グラフ。LLM が設計戦略を関係クエリ化 → Cypher 風照合でサブ回路/テストベンチ取得 → 局所注釈で組み立て。**検索結果＝答えでなく組み立て部品（grounding）**。
- **数値**`[本文のみ]`: 894 回路、部品検出 recall 97%、生成 96%。← **要旨に無い。本体表で要確認。**
- **TopoRAG への含意（＋）**: 「検索＝根拠」を回路で実証（向きは生成）。TopoRAG が認識でカード＝根拠を採るのは整合。
- **含意（−）**: 規模（894）・GPT-4 依存・閉じた注釈型回路群。TopoRAG（~47・決定的）とは前提が違い、ここから TopoRAG の手法的優位は導けない。

### 2.2 TopoSizing（arXiv:2509.14169, 提出 2025-09-17）`[要旨確認]` — 階層グラフ＋LLM 理解（認識に最も近い）
- **方法**`[要旨確認]`: グラフアルゴリズムで **device–module–stage** の3層へ組織化 → LLM エージェントが**仮説→検証→精緻化ループ＋組み込み整合性チェック**で注釈生成 → 洞察を **Bayesian 最適化**へ統合。
- **検索**: 明示的な外部 RAG は取得範囲では確認できず。
- **タスク**: 理解＋**サイジング**（純認識ではない）`[要旨確認]`。
- **TopoRAG への含意（＋）**: 「決定的に階層化 → LLM が理解」は TopoRAG の目標とほぼ同型。階層 IR の作り方が直接参考になる。
- **含意（−）**: 最終目的がサイジングで、TopoRAG の認識＋棄却とは出口が違う。「同型だから正しい」とは言えない。
- ※評価数値は圧縮PDFで未取得。

### 2.3 HeaRT（arXiv:2511.19669, 提出 2025-11-24）`[要旨確認]` — 階層推論ツリー
- **要旨で確認**: 40 回路 AMS ベンチで、few-shot 比 **F1(subcircuits) を ≥13.5%、F1(loops) を ≥37.8% 改善**。タスクは設計最適化。
- **本文のみ**`[本文のみ]`: device-net 二部グラフ→電源からの多源 BFS で DC 連結成分→LLM CoT 分解。知識DBは「正規化グラフ＋メトリック順位＋**WL 指紋をハッシュ索引**＋文献根拠」。検索は**順位ベース**でベクトル類似ではない。難回路で **F1 が 0.336 程度まで低下**。
- **TopoRAG への含意（＋）**: WL を構造同一性の核に使う点が TopoRAG と符合 `[本文のみ]`。知識DB＝「正規化グラフ＋根拠」は知識カードの上位テンプレ。**階層化が few-shot を上回る**（≥13.5%）のは構成的認識を後押し `[要旨確認]`。
- **含意（−／反証）**: それでも**難回路で絶対 F1 は低い**（0.336, `[本文のみ]`）。「ブロック単位にすれば解決」ではなく、**構成的にしても難回路は依然難しい**。TopoRAG の処方箋は過信禁物。

### 2.4 CIRCUIT（arXiv:2502.07980, 提出 2025-02-11）`[要旨確認]` — LLM 回路理解の外部ベンチ
- **要旨で確認**: **510 QA**。最良 **GPT-4o 48.04%**（最終数値）。**unit test は 27.45% しか通らない**（＝同一トポロジーの数値変動に弱い）。
- **本文のみ**`[本文のみ]`: 102 テンプレ×5 設定、Basic/Analog/Power/RF、図/netlist 併用、Level1 48.5%→Level5 9.2%、GPT-4-turbo/Gemini1.5 も評価、netlist を与えても改善限定。
- **TopoRAG への含意（＋）**: TopoRAG の **Stage 2 評価が文字列マッチ1個**という弱点への**既製の物差し**。unit-test 式（pass@k/n）と誤り分類は採用候補。
- **含意（−）**: CIRCUIT は数値解析タスク中心で、TopoRAG の「構成同定」とは問う内容がずれる。そのまま流用はできず翻訳が要る。

### 2.5 AnalogRetriever（arXiv:2604.23195, 提出 2026-04-25, 著者 Wang ら）`[要旨確認]` — クロスモーダル回路検索
- **要旨で確認**: 6 方向クロスモーダル検索で **平均 R@1 = 75.2%**。
- **本文のみ**`[本文のみ]`: netlist=Port-Aware R-GCN（端子種別を区別）／図・文=CLIP→共有埋め込み、三モーダル対照学習。データ修復・19 クラス注釈に LLM 使用。
- **訂正（旧版の誤り）**: 旧版に「GPT-5.4 を使用」と書いたが、**要旨に該当なく一次確認できないため削除**。具体モデル名は未確認。
- **TopoRAG への含意（＋）**: 端子種別を区別するグラフ表現は TopoRAG の端子命名規則と同方針。R@1≈75% は「実機検索の相場観」の参考。
- **含意（−）**: 閉集合（19 クラス）で OOD 棄却なし。学習埋め込み＋大規模前提で、TopoRAG の小規模・決定的路線とは別物。

### 2.6 GANA / GCN 系（DATE'20, TCAD'23）`[要約のみ]` — LLM 以前の認識中核
- **要約のみ**: フラット netlist をグラフ化し GCN＋近似部分グラフ同型でサブ回路同定、対称/整合制約も検出。LLM/RAG 不使用。
- **重要な但し書き**: **全文未取得（証明書エラー）**。方法の具体（「近似部分グラフ同型」等）は検索スニペット由来で**未検証**。
- **TopoRAG への含意（＋／−）**: 「サブ回路単位の構成的認識」は LLM 以前から確立、という主張の傍証になり得るが、**根拠が弱い**ため強くは依拠しない。

**含意（−／新規性の否定）**:
1. **分類の枠は完全に既存**。特に MDPI（GCN で少数アナログ回路種を分類）は貴手法の検索段と看板がそのまま重なる。ASIC-GNN は 98.3%、CktGNN は 10K 規模と、**学習ベースは規模・精度で貴手法（手設計 43 次元・DB47・最近傍）を上回る側**にいる。貴手法自身の README も検索段を「フラット最近傍分類器＝素朴」「特徴量はパッチの塔（スケールしない）」と認めている。

## 3. TopoRAG の位置付け（肯定・否定・不確実を分離）

### 3.1 主流と一致している点（追い風）
- 決定的な構造前処理（IR）重視 ← TopoSizing の device-module-stage `[要旨確認]`。
- WL の採用 ← HeaRT も WL を使う `[本文のみ]`（要旨では未確認のため傍証どまり）。
- 端子の意味を区別 ← AnalogRetriever の Port-Aware 表現 `[本文のみ]`。
- 「検索＝根拠」への移行 ← AMSnet-KG の思想 `[本文のみ]`。

### 3.2 「空き」に見える点 ―― ただし証明ではない
読んだ数本の中では、**(a) 認識特化、(b) OOD 棄却、(c) 小規模・決定的** が見当たらない。しかし：
- これは **便宜標本での観察**であって「空きの証明」ではない（無いことの証明には系統網羅が要る）。
- **OOD/オープンセット認識自体は成熟した大分野**（OpenMax 等、§5 参照）。仮に新規性があるとすれば「OSR そのもの」ではなく「**回路認識に OSR を持ち込み、事前登録ゲートで評価する運用**」という限定的な角度。
- 回路特化の OOD 研究を**私はまだ検索しきれていない**（1 クエリのみ、回路特化はヒットせず＝存否不明）。**主張は「暫定的観察」に格下げする。**

> **【2026-07-08 追記】上の (a)(b)(c) は、分類軸の先行研究（§2.7）と実験Aの結果を踏まえて次のように格下げする**：
> - **(構造特徴による)分類 は「空き」ではない**。§2.7 のとおり成熟領域で、貴手法はむしろ規模・精度で見劣りする側。新規性の核にはできない。
> - **(b) OOD 棄却も「棄却できること」自体は差別化にならない**。実験A（`RESULTS_experiment_A_2026-07-05.md`）で直読み LLM の out 棄却率 95.0% が提案手法 90.0% を**上回った**。棄却"率"では負けている。
> - **残るのは「監査可能・再現可能・根拠付き（決定的 IR＋弁別カード）の認識」という運用面の非対称のみ**。実験A の P2（直読みは閉集合を外すと棄却率 5%＝検証可能なラベル空間・決定的棄却に載らない）が、これを支える唯一の明確な証拠。**発表の新規性主張はこの 1 点に集約すべき**。ただしこれは「手法の新規性」ではなく「運用・保証の新規性」であり、会場によっては弱いと自覚しておく。

### 3.3 弱点（改善余地）と、その反証
- **フラット最近傍は素朴**。主流（TopoSizing/HeaRT/GANA）はサブ回路単位。`block_decomposer` を持ちながら 43 次元に潰しているのは後退。
  - **反証**: HeaRT は構成的でも**難回路で F1 0.336**`[本文のみ]`。構成化は方向として妥当でも**万能ではない**。TopoRAG で導入しても難回路は残ると見積もるべき。
- **知識の持ち方が薄い**（知識カードは初期段階）← HeaRT の知識DB（正規化グラフ＋順位＋根拠）が上位テンプレ `[本文のみ]`。
- **Stage 2 評価が貧弱** ← CIRCUIT の unit-test 式が既製の物差し `[要旨確認]`。

---

## 4. 示唆（次アクション）― 反証を織り込んだ上で

1. **構造**: `block_decomposer` 出力を活かし**ブロック単位の照合**を追加。フラット最近傍は OOD 用補助へ降格。
   ※ただし HeaRT の反証（難回路で低 F1）を踏まえ、**「構成化＝解決」ではなく「候補出しの質向上」**として控えめに期待値設定する。
2. **知識**: 知識カードに WL 指紋・関係（is-a/composed-of/implies-function）・文献根拠を持たせ、「答え」でなく「根拠」を返す KB へ（HeaRT 流）。
3. **評価**: Stage 2 に CIRCUIT 流 unit-test（pass@k/n）と誤り分類を導入し、自前評価を学術ベンチと比較可能に。OOD 棄却は独自軸として維持。
4. **論文上の positioning**: 「認識特化＋オープンセット棄却」を狙うなら、**まず系統的サーベイで空きを実証してから**主張する（現状は暫定）。

## 5. 「空き」主張の検証メモ（OOD/OSR）

OOD/open-set 検索（1 クエリ）では**回路特化の OOD 研究は surface せず**、汎用 OSR/OOD 文献のみ（OpenMax, OSR/OOD サーベイ等）。含意:
- OSR/OOD は成熟分野。TopoRAG の棄却は**手法的に新規ではない**（既存 OSR の回路応用）。
- 回路特化 OOD の**存否は未確定**。「空き」を論文主張に使うには、`circuit topology open-set / novelty detection / unknown subcircuit` 等で**追加の系統検索が必須**。
